Take-Home Assignment

Junior IT Developer & Systems Specialist — United Rental Group

Time guidance: This task should take approximately 1–2 hours. Please don't spend longer than that — we'd rather see a working prototype with clear thinking than a polished product.

Deadline: Please return your submission within 5 working days of receiving this brief.

The Scenario

A fictional vehicle rental company operates a fleet of vehicles across multiple branches. They currently have no visibility into their stock — the data sits in a spreadsheet and nobody can quickly answer basic questions like "how many vehicles are available right now?" or "which branch has the most vehicles in maintenance?"

Your job is to build a simple web-based dashboard that turns this data into something useful.

What You're Given

Attached to this brief is a CSV file (fleet_sample_data.csv) containing approximately 100 rows of sample fleet data with the following columns:

  • vehicle_id — unique identifier
  • make — manufacturer (e.g. Ford, Vauxhall, Toyota)
  • model — vehicle model
  • year — year of registration
  • fuel_type — petrol, diesel, electric, or hybrid
  • mileage — current odometer reading
  • branch — which branch the vehicle is assigned to
  • status — one of: available, on_rent, or maintenance
  • daily_rate — rental rate in £ per day
  • last_service_date — date of most recent service

Your Task

  1. Build a simple web dashboard that displays the fleet data in a useful way.
    • Use whatever technology you're comfortable with — React, Vue, plain HTML/JavaScript, Python with Flask, or anything else. Your choice.
    • It doesn't need to be beautiful. It needs to be functional and usable.
  2. Include at least the following:
    • A summary view showing: total vehicles, how many are available / on rent / in maintenance, and a breakdown by branch
    • A way to filter or search the data (e.g. by branch, status, fuel type, or similar)
    • One simple visualisation — a chart or graph of your choice (e.g. vehicles by status, mileage distribution, branch comparison)
  3. Use an AI coding assistant to help you build it.
    • You can use Claude, ChatGPT, GitHub Copilot, or any similar tool — your choice
    • This is not optional. We specifically want to see how you work with AI tools. It's a core part of the role.
  4. Write a short README (maximum 1 page) covering three things:
    • What you built and why you made the technical choices you did
    • How you used the AI assistant — what it helped with, what didn't work, and what you had to fix or do yourself
    • What you'd improve or add if you had more time

What to Submit

Option A: A link to a GitHub repository containing your code, the README, and instructions on how to run it.

Option B: A zip file containing all of the above, emailed back to us.

What We're Looking For

There is no single right answer. We're interested in how you think and how you work. Specifically:

  • Does it work? Can we follow your instructions and see a functioning dashboard?
  • Is your code readable? Clean structure, sensible naming, no unnecessary complexity.
  • How did you use AI? The README section on this matters. We value honesty — tell us what the AI did well and where you had to step in.
  • Can you communicate clearly? Is the README well-written and easy to follow?
  • Did you make smart trade-offs? With limited time, what did you prioritise and what did you leave out? Good judgement matters more than feature count.

If you have any questions about the brief, please don't hesitate to ask. Good luck — we're looking forward to seeing what you build.
